# SYF.ANTYDIZAJN.PL

> Publiczny hosting plików Markdown z brutalnie szczerym podejściem.

---

## 🎯 OVERVIEW

**URL:** `syf.antydizajn.pl`
**Stack:** Next.js 16.1.1 (App Router, Turbopack)
**Cel:** Wrzucasz plik `.md` do folderu `files/` → automatycznie dostępny pod `/NAZWA`

---

## 🎨 LAYOUT REFERENCE (KRYTYCZNE!)

> **UŻYJ DOKŁADNIE 1:1 LAYOUTU Z FOLDERU `layout/`**

Folder `syf/layout/` zawiera gotowy prototyp HTML/CSS/JS który MUSI być użyty jako baza dla implementacji Next.js:

```
layout/
├── index.html      # Strona główna - lista plików
├── file.html       # Strona pojedynczego pliku MD  
├── about.html      # Strona "O SYF" / manifest
├── styles.css      # Główne style (CSS variables, overlays, animacje)
├── markdown.css    # Style renderowania MD (neonowe kolory)
└── script.js       # Interaktywność (dropdown, glitch, cursor glow)
```

**ZASADY:**
1. **Kolory, fonty, efekty** - przenieś 1:1 z `styles.css`
2. **Markdown rendering** - użyj stylów z `markdown.css`
3. **Struktura HTML** - zachowaj strukturę komponentów
4. **Animacje** - scanlines, glow orbs, glitch effect - wszystko zostaje
5. **NIE ZMIENIAJ** designu - to jest finalna wersja wizualna

## 🔧 FUNKCJONALNOŚCI

### 1. Strona główna (`/`)
- **Lista wszystkich plików** z folderu `files/`
- **Sortowanie:** 
  - Data modyfikacji (domyślne, najnowsze najpierw)
  - Data modyfikacji (najstarsze)
  - Nazwa A-Z
  - Nazwa Z-A
- **Paginacja:** 30 plików na stronę
- **Wyświetlane info:**
  - Nazwa pliku (bez .md)
  - Data modyfikacji
  - Rozmiar pliku
  - Opcjonalnie: pierwszy paragraf jako preview

### 2. Strona pliku (`/[slug]`)
- **Renderowanie MD** w stylu jak na screenshocie:
  - Ciemne tło (#0d0d0d lub podobne)
  - Magenta/różowe nagłówki H1, H2
  - Cyan/turkusowe H3, H4
  - Żółte/pomarańczowe separatory/linie
  - Czerwone bullet points
  - Zielone linki i kod inline
  - Monospace font (JetBrains Mono / Fira Code)
- **Pasek na górze:** "Download as: MD | PDF | HTML | TXT | DOCX"
- **Metadata:** data utworzenia, data modyfikacji

### 3. API Download (`/api/download/[slug]`)
Query param: `?format=md|pdf|html|txt|docx`

**Formaty:**
- **MD** - oryginał, bez zmian
- **PDF** - wygenerowany z puppeteer/html-pdf, zachowuje style
- **HTML** - standalone z inline CSS, gotowy do otwarcia w przeglądarce
- **TXT** - plain text, stripped z formatowania
- **DOCX** - dla korporatów (pandoc lub docx lib)

## 🎨 STYL RENDEROWANIA MD

Bazując na screenshocie:

```css
/* Przykładowe mapowanie */
--bg-primary: #0d0d0d;
--bg-secondary: #1a1a1a;
--text-primary: #e0e0e0;
--heading-h1: #ff00ff; /* magenta */
--heading-h2: #ff1493; /* deep pink */
--heading-h3: #00ffff; /* cyan */
--separator: #ffa500; /* orange */
--bullet: #ff4444; /* red */
--link: #00ff00; /* green */
--code-inline: #00ff88;
--code-block-bg: #1e1e1e;
--font-mono: 'JetBrains Mono', 'Fira Code', monospace;
```

**Elementy specjalne:**
- Nagłówki: uppercase, letter-spacing
- Separatory: grube kolorowe linie
- Blockquote: kolorowy border-left
- Code blocks: syntax highlighting (Prism/Shiki)
- Listy: kolorowe bullets/numeracja
